    Sustainable Innovation Camp is a recurring event organized around Denmark by Sustainary for university students and recent graduates, who work in teams to tackle real-world problems that require innovative and sustainable solutions.

    By signing up for a camp, the participants take part in a series of workshops on how sustainability is shaping business, followed by a 3 to 5 days case-competition in the scope of sustainable innovation.

    My contribution for this event was to design and create a website, which included a landing page, separate pages for each camp, and a form to sign up for a camp.

    I made it very easy to update this website by creating a template for a new camp, which can simply be copied and quickly filled out, and subsequently hooked to the rest of the wesbite.

    Moreover, I have created a single source of truth for all the information about all the camps. The status of a camp can be either upcoming or past, depending on if it is about to take place or if it has already taken place. This status is changed in one place and updated automatically throughout the whole website without any manual intervention (camps are automatically sorted in the Camps page, and they are added or removed from the sign up form on their own).

    Finally, I have used a wide array of technologies to develop this website, mainly React, TypeScript, and Tailwind CSS for creating separate pages and components, ensuring type safety, and styling. React Router DOM was used for client-side routing, and meta tags were inserted into each page with the help of React Helmet Async. Server-side logic for validating data and sending emails was initially written in Node.js, but was later rewritten in Next.js in order to deploy the project on Vercel. Sending emails was handled using SendGrid, and the code was tracked by Git and hosted on GitHub. Last but not least, code quality and consistent formatting was assured with ESLint and Prettier.

    This endless runner game was inspired by the Dinosaur Game, which runs in Google Chrome when there is no internet connection.

    The gameplay is straightforward, your job is to jump over as many cacti as possible and get as many points as you can. To jump, press the Spacebar on your keyboard or tap the screen on your touchscreen device. To pause the game, press the Pause Game button or the Escape key.

    This JavaScript game is packed with details and my goal was to make the user experience smooth, and for the environment to feel alive, mainly by using lots of animations happening simultaneously (cacti moving, clouds floating, player jumping and blinking, shadow and score resizing). This led to many consideration in terms of performance. The design is all done is CSS and sound effects are all created by me.
